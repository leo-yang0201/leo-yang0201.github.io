Leetcode 104 Maximum Depth of Binary Tree（二叉树的最大深度）

Posted on 2020-01-27 Edited on 2020-03-10 In leetcode

Difficulty : Easy

Type: Tree && Depth-first Search

Language : Java

Clarification

Given a binary tree, find its maximum depth.

The maximum depth is the number of nodes along the longest path from the root node down to the farthest leaf node.

Note: A leaf is a node with no children.

Possible Solutions

  • 递归
    DFS思想一次性找到左右子树的深度，比较左右子树深度大小，返回，因为每个结点只访问一次，时间复杂度O(N)，N是结点的数量。空间复杂度在最坏情况下是O(N)，即每个结点只剩下左结点，最好情况下（树完全平衡），空间复杂度O(log(N))
  • 迭代
    BFS思想，记录二叉树的层数，遍历完，层数为最大深度。时间复杂度O(N)，使用队列实现。

Test Cases

example:

return its depth = 3.

Leetcode 225 Implement Stack with Queue(用队列实现栈)

Posted on 2020-01-22 Edited on 2020-03-05 In leetcode

Difficulty : Easy

Type: Stack,Design

Language : Java

  1. Clarification

    • Implement the following operations of a stack using queues.
      • push(x) – Push element x onto stack.
      • pop() – Removes the element on top of the stack.
      • top() – Get the top element.
      • empty() – Return whether the stack is empty.

    Notes:

    • You must use only standard operations of a queue – which means only push to back, peek/pop from front, size, and is empty operations are valid.
    • Depending on your language, queue may not be supported natively. You may simulate a queue by using a list or deque (double-ended queue), as long as you use only standard operations of a queue.
    • You may assume that all operations are valid (for example, no pop or top operations will be called on an empty stack).

  2. Possible Solutions

    • 出队入队操作
      入队时，先把当前值入队，再把顺序倒成栈的样子，具体操作是将当前出队的值重新入队，这个值就会从队列头移到队列尾，每次移动队列长度 - 1 次。

  3. Code

  4. Test Cases

    MyStack stack = new MyStack();

    stack.push(1);
    stack.push(2);
    stack.top(); // returns 2
    stack.pop(); // returns 2
    stack.empty(); // returns false

Language : Java

  1. Clarification

    Implement the following operations of a queue using stacks.

    • push(x) – Push element x to the back of queue.
    • pop() – Removes the element from in front of queue.
    • peek() – Get the front element.
    • empty() – Return whether the queue is empty.

    Notes:

    • You must use only standard operations of a stack – which means only push to top, peek/pop from top, size, and is empty operations are valid.
    • Depending on your language, stack may not be supported natively. You may simulate a stack by using a list or deque (double-ended queue), as long as you use only standard operations of a stack.

  2. Possible Solutions

    • 双栈解法
      一个输入栈，一个输出栈，数据push时进输入栈，输出的时候只要把输入栈的数据入栈输出栈，再pop输出栈就倒过来成队列了

  3. Code

  4. Test Cases

    MyQueue queue = new MyQueue();

    queue.push(1);
    queue.push(2);
    queue.peek(); // 返回 1
    queue.pop(); // 返回 1

Leetcode 160 Intersection of Two Linked Lists(相交链表)

Posted on 2020-01-19 Edited on 2020-03-05 In leetcode

Difficulty : Easy

Type: Linked List

Language : Java

  1. Clarification

    Write a program to find the node at which the intersection of two singly linked lists begins.

    Notes:

    • If the two linked lists have no intersection at all, return null.
    • The linked lists must retain their original structure after the function returns.
    • You may assume there are no cycles anywhere in the entire linked structure.
    • Your code should preferably run in O(n) time and use only O(1) memory.

  2. Possible Solutions

    • 双指针
      这道题因为要求用O(1)的时间复杂度，所以不能考虑新建一个Set，存第一个链表的值，再遍历下一个链表找Set中对应值的方法。设n1,n2两个指针，分别放在headA和headB，只遍历一次，如果n1走到null，就把n1放到headB，否则n1继续走，如果n2走到null，就把n2放在headA，否则n2继续走，因为两个链表的长度可能存在差值，先走到null那个指针放回另一个链表的头部后，两个指针只要持续往下走就一定会相交，交点就是这两个链表的交点（思路类似环形链表，只不过是人工进行了环形操作）

  3. Code

  4. Test Cases

    Input: intersectVal = 2, listA = [0,9,1,2,4], listB = [3,2,4], skipA = 3, skipB = 1
    Output: Reference of the node with value = 2
    Input Explanation: The intersected node’s value is 2 (note that this must not be 0 if the two lists intersect). From the head of A, it reads as [0,9,1,2,4]. From the head of B, it reads as [3,2,4]. There are 3 nodes before the intersected node in A; There are 1 node before the intersected node in B.

    Input: intersectVal = 0, listA = [2,6,4], listB = [1,5], skipA = 3, skipB = 2
    Output: null
    Input Explanation: From the head of A, it reads as [2,6,4]. From the head of B, it reads as [1,5]. Since the two lists do not intersect, intersectVal must be 0, while skipA and skipB can be arbitrary values.
    Explanation: The two lists do not intersect, so return null.
