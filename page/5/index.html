Leetcode 5 Longest Palindromic Substring(最长回文子串)

发表于 2019-12-26

Difficulty : Medium

Type: String, Dynamic Programming

Language : Java

  1. Clarification

    Given a string s, find the longest palindromic substring in s. You may assume that the maximum length of s is 1000.

  2. Possible Solutions

    • 暴力破解法（Brute Force）

      1. 枚举所有长度大于等于2的子串，依次判断它们是否为回文
      2. 此方法会超时，无法通过AC，可用于检验思路

    • 中心扩散法

      1. 回文串就是从中间开始往两边走，字符是轴对称的，根据此特性生成字符串，一共有2n-1个中心，奇数长度时中心个数是n个，偶数长度时中心个数是n-1个，所以一共2n-1个中心
      2. expandByCenter方法从中间点往两边找最长的回文串，比较长度，记录最长长度的start index和end index，作为返回substring的index

  3. Code

时间复杂度：O(n^3) 空间复杂度O(1)

  1. Test cases

    Input: “ababcba”
    Output: “abcba”

Leetcode 25 Reverse Nodes in k-Group(k个一组翻转链表)

发表于 2019-12-18

Difficulty : Hard

Type: Linked list

Language : Java

  1. Clarification

    Given a linked list, reverse the nodes of a linked list k at a time and return its modified list. k is a positive integer and is less than or equal to the length of the linked list. If the number of nodes is not a multiple of k then left-out nodes in the end should remain as it is.

  2. Possible Solutions

    • Non-recursive

      Set a dummy code in the beginning and we need prev and last pointer. Last pointer will move k*1 distance. If k = 3, then last is 4. If last is null , then left nodes are smaller than k, which means these nodes are not supportive for next swap. Each iteration we reverse elements between prev and last (last and prev are excluded). For example, k = 3, we reverse elements from 1 to 3:

      1. We put 2 after prev and connects it to 1. Then linked list becomes prev -> 2 -> 1-> 3 -> last
      2. We put 3 after prev and connects it to 2. Then linked list becomes prev -> 3 -> 2 -> 1 -> last

Pointer tail will point to 1 in this case. ( Tail’s next node will always be Last! ). Tail will become the prev for next swap.

  1. Code

    Space Complexity : O(1)

  2. Test Cases

    Given this linked list: 1->2->3->4->5

    For k = 2, you should return: 2->1->4->3->5

    For k = 3, you should return: 3->2->1->4->5

Leetcode 142 Linked List Cycle ||

发表于 2019-12-17

Difficulty : Medium

Type: Linked List && Two Pointers

Language : Java

  1. Clarification

    Given a linked list, determine if it has a cycle in it. Return the node where the cycle begins.

  2. Possible Solutions

    • HashSet
      If hashset contains node, it means that the current node is the entrance of the cycle. We can return head thereby.

    • Faster and Slower Pointers
      In this case, let slow and fast pointers start from the head together. We know that fast pointer moves two steps each time and slow moves one step. Eventually they will encounter in the cycle. Then move the fast pointer to the head and set its speed to one step each time. Fast and Slow pointer will meet at the entrance of cycle this time. Return Slow pointer then.

      Why they will encounter at the entrance at second time? Because fast pointer will move N*cycle length distance then slow. So, non-cycle length + distance between the entrance of cycle to meet point = N * cycle length.

  3. Code

HashSet’s runtime is slower than Fast and Slow Pointers because new a hashset will cost extra time. It doesn’t mean that this algorithm’s speed is lower than Fast and Slow Pointers.

  1. Test Cases

    Input : head = [3,2,0,-4], pos = 1

    Output : tail connects to node index 1

Leetcode 141 Linked List Cycle(环形链表)

发表于 2019-12-17

Difficulty : Easy

Type: Linked List && Two Pointers

Language : Java

  1. Clarification

    Given a linked list, determine if it has a cycle in it.

  2. Possible Solutions

    • HashSet
      Iterate all the nodes and use HashSet to store references of nodes. If head is not null, determine whether HashSet contain head. If not, add head into Set. After each interation, head = head.next
    • Faster and Slower Pointers
      In order to determine whether a linked list contains circle, we can set two pointers. One is slow pointer that only moves a step each time and another is fast pointer that moves two steps each time. If the list has circle, fast pointer will equal to slow pointer eventually.

  3. Code

HashSet’s runtime is slower than Fast and Slow Pointers because new a hashset will cost extra time. It doesn’t mean that this algorithm’s speed is lower than Fast and Slow Pointers.

  1. Test Cases

    Input : head = [3,2,0,-4], pos = 1

    Output : true

    Input : head = [1,2], pos = 0

    Output : true

    Input : head = [1], pos = -1

    Output : false

Leetcode 24 Swap Nodes In Pairs(两两交换链表中的节点)

发表于 2019-12-16

Difficulty : Medium

Type: Linked List

Language : Java

  1. Clarification

    Given a linked list, swap every two adjacent nodes and return its head
    You may not modify the values in the list’s nodes, only node itself can be changed.

  2. Possible Solutions

    • Iteration

      Set a node,which value is zero, called pre and pre.next points to head.
      Set another node temp equals to pre.

      Set two nodes start and end. temp.next points to start, temp.next.next points to end. In this way, there are only one node or zero node left if start and end are null.
      Basically, we choose three nodes as a group and swap the last two and then we set the new temp to start.
      So, temp.next(pre) -> end(2). start.next(1) -> end.next(3). end.next(2) -> start(1). And now the linked list is pre -> 2 -> 1 -> 3 -> 4 -> null. Set temp = start(1) and begin new swap.

  • Recursion
    This is much tricky. We have to find three important factors to analyze recursion ^_^

    1. When to terminate?
      head is null or head.next is null. It will terminate when there is only one node or no node.
    2. What’s the repetitive action?
      head.next -> next.next
      next.next -> head
      This is the process of swaping.
    3. Return value
      return value is next pointer because next comes before head after swaping.

  1. Code

  1. Test Cases

Given 1->2->3->4, return the list as 2->1->4->3.

Leetcode 206 Reverse Linked List(反转链表)

发表于 2019-12-16

Difficulty : Easy

Type: Linked List

Language : Java

  1. Clarification

    Reverse a singly linked list

  2. Possible Solutions

    • Iteration
      Set three LinkNode prev, curr and temp. prev and temp are null at first and curr equals to head. prev is the pointer that point to last node. Each time we make curr.next point to previous node and set the temp to curr.next, prev to curr, curr to temp. So that each time the node in linked list will point to previous node and reverse the whole list in the end.
    • Reversion
      head.next.next=head means that head.next point to null at the first and add a next pointer to point itself. So, head.next.next point to head. (So hard to explain it by words - -. There is an animation to show the process)

  3. Code

  1. Test Cases

    输出: 5->4->3->2->1->NULL

Difficulty : Medium

Type: Array && Two Pointers

Language : Java

  1. Clarification

    Find all unique triplets in the array which gives the sum of zero

    No duplicate triplets

  2. Possible Solutions

    • Two Pointers
      This question actually use three pointers. The result is to find all pairs of value a and b to make sure a+b=-c. Another pointer is used to record the c.

      The solution is to sort the array in the beginning. And then run through all indices of a possible first element of a triplet. For each possible first element, find the indices for last two elements that make sure their sum is the negative first element.

      Remember to skip equal elements to avoid duplicates in the answer.

  3. Code

  1. Test Cases
    nums = [-1, 0, 1, 2, -1, -4]

    res = [ [-1,0,1],[-1,-1,2]]
