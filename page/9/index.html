Statistics for IT Managers 95-796 notes（知识点汇总）

Posted on 2020-01-02

  1. Descriptive Statistics and Probability

    • Basic Definitions 基础概念

      • Statistics is the science of analyzing and interpreting data, transforming raw data into information.
      • Descriptive statistics 描述统计
        • Graphical displays (histograms, box plots) 图表形式呈现
        • Numerical summaries(mean , medium, mode, variance) 数据汇总
      • Inferential statistics 推断统计
        • use sample data to make estimates, decisions, predictions, or other generalisations about a large set of data. 利用样本数据推断总体特征的统计方法

    • Data types 数据类型

      • Qualitative(or categorical) data: each data point is classified into one of a given set of categories.
        • Nominal data 分类数据
        • Ordinal data 顺序数据
      • Quantitative(or numerical data) : each data point is measured on a naturally occurring numerical scale 数据值数据
        • Height, weight, income, etc.

    • Numerical descriptive statistics

      • Measures of the center of the data
        • Mean 平均值, median 中位数, mode 众数
      • Measures of variability
        • Variance 方差，standard deviation 标准差, range 极差, interquartile range 四分位差

    • Skewed distribution

      • Symmetric: mean = median
      • Positively skewed: mean > median
      • Negatively skewed: mean < median

    • The empirical rule 经验法则（前提：对称分布）

      • 68% of the measurements will fall within 1 standard deviation of the mean
      • 95% of the measurements will fall within 2 standard deviation of the mean
      • 99.7% of the measurements will fall within 3 standard deviation of the mean
      • 该规则用于识别outliers（离群点），

    • Probability 概率

      • Basic definitions

        • P(A) >=0 && P(A) <= 1
        • P(～A) =1-P(A)， P(~A)也可以表示为P(A’)
        • Proabilities can be objective or subjective

      • Combining probablilities

        • P(AUB)=P(A)+P(B)–P(A∩B),若A与B互斥(Mutually exclusive)，则P(AUB)=P(A)+P(B)

      • Conditional probailities

        • P(A | B) = P(A ∩ B) / P(B) , P(B) > 0

      • Independent events

        • 两个事件独立的条件：P (A | B) = P(A | ~B) = P(A), and P(B | A) = P(B | ~A) = P(B)，一个事件发生与否不影响另一个事件发生的概率
        • 如果事件A，B相互独立：P(A ∩ B) = P(A | B) P(B) = P(A) P(B)

      • Bayes’ Theorem （贝叶斯公式，逆概率公式）

        • P(A |B)=P(B|Ai)P(Ai) / P(B|A )P(A )+…+P(B|A )P(A)
        • P(A) is prior proabilities为原因A的先验概率，P(A|B) 后验概率

      • Random variables

        • Sample space 样本容量
        • Random variable 随机变量
          • 离散型随机变量(discrete)
            • We must have p(x)≥0 for all x,and ∑p(x) = 1
            • Mean(or expected value):μ=∑x*p(x).
            • Variance: σ^2 = ∑(x – μ)^2 p(x)
          • 连续型随机变量(continuous)
            • P(a ≤ x ≤ b) = ∫f(x)dx
            • f(x) >= 0 for all x and ∫f(x)dx = 1
            • Mean: μ = ∫x f(x)dx
            • Variance: σ = ∫(x − μ) f(x)dx

      • The binomial distribution 二项分布

        • 成功概率为P，失败概率为1-P

        • P(x) =( n! / x!(n-x)! ) * p^x(1-p)^x, for 0<= x <= 1

        • For x ~ Bin(n,p)
          Mean of x: μ = np. Variance of x: σ^2 = np(1-p)

      • The uniform distribution 正态分布

        • if x~Uniform(c,d), when c <= x <= d, f(x) = 1/(d-c), otherwise f(x) is 0
        • Mean: μ = (c + d) / 2
        • Variance: σ^2 = (d – c)^2 / 12

      • Computing Z-scores

        • If x is distributed according to N(μ, σ), then z = (x − μ) / σ will be distributed according to the

          standard normal distribution, N(0,1)

      • Using $\bar{x}$ to estimate μ

        • If the population is normally distributed with mean μ and standard deviation σ, then the sample mean x is also normally distributed, with mean μ and standard deviation σ / N .
        • Central Limit Theorem 中心极限理论
          • If the population has any distribution with mean μ and standard deviation σ, and if N ≥ 30, then the sample mean x is normally distributed, with mean μ and standard deviation σ / N .

Leetcode 243 Palindrome Linked List(回文链表)

Posted on 2019-12-29 Edited on 2020-03-05 In leetcode

Difficulty : Easy

Type: Linked List, Two Pointers

Language : Java

  1. Clarification

    Given a singly linked list, determine if it is a palindrome. 要求使用O(n)时间复杂度和O(1)空间复杂度

  2. Possible Solutions

    • 快慢指针
      1. 回文是轴对称的，所以思路是先找到轴，然后把轴右侧的链表进行反转，再遍历比较两个链表的值是不是一致，都是一致的就是回文
      2. 快指针每次走两个，慢指针每次走一个，这样快指针为空或者快指针的下一个结点为空时，慢指针正好走到轴心
      3. 此时慢指针在中心点，破坏链表，让慢指针的下一个结点指向null，再把慢指针放回到第一个结点处，为后续遍历比较作准备
      4. 把后半段的链表进行反转，方法是设置一个空结点pre，先用temp结点储存当前结点cur的下一个结点，再让cur指向pre，再让pre=cur，cur=temp，最后pre结点就是反转后链表的头结点
      5. 比较slow结点和pre结点的值是否相等，相等就继续迭代（注：结点数为奇数时，左边会比右边多一个结点，所以从右边开始迭代），都相等就返回true，否则false

  3. Code

  1. Test Cases

    Input: 1->2->2->1
    Output: true

Leetcode 641 Design Circular Deque(设计循环双端队列)

Posted on 2019-12-29 Edited on 2020-03-05 In leetcode

Difficulty : Medium

Type: Design, Queue

Language : Java

  1. Clarification

    • Design your implementation of the circular double-ended queue (deque).

      Your implementation should support following operations:

      • MyCircularDeque(k): Constructor, set the size of the deque to be k.
      • insertFront(): Adds an item at the front of Deque. Return true if the operation is successful.
      • insertLast(): Adds an item at the rear of Deque. Return true if the operation is successful.
      • deleteFront(): Deletes an item from the front of Deque. Return true if the operation is successful.
      • deleteLast(): Deletes an item from the rear of Deque. Return true if the operation is successful.
      • getFront(): Gets the front item from the Deque. If the deque is empty, return -1.
      • getRear(): Gets the last item from Deque. If the deque is empty, return -1.
      • isEmpty(): Checks whether Deque is empty or not.
      • isFull(): Checks whether Deque is full or not.

  2. Possible Solutions

    • 取余实现循环双端队列
      • 思路同leetcode 622（设计循环队列），区别在于此时数据可以从队列头入队/出队，也可以从队列尾入队/出队，实际上双端队列是栈和队列的结合。（既可以后入先出，也可以先入先出）
      • 队列头front指向队列头部第1个有效数据的位置
      • 队列尾rear指向队列尾部（即最后一个有效数据）的下一个位置，即下一个从队尾入队元素的位置
      • 为了避免“队列为空”或者“队列为满”的判别条件冲突，有意浪费了一个位置
        • 浪费一个位置是指：循环数组中任何时刻一定至少有一个位置不存放有效元素
        • 判别队列为空的条件是 rear == front，判别队列为满的条件是 (rear + 1) % capacity == front，可理解为当rear循环到数组的前面，要从后面追上front，还差一格的时候，判断队列为满
      • 注意数组下标可能越界的情况
        • 指针后移的时候，索引 + 1，要取模
        • 指针前移的时候，为了循环到数组的末尾，需要先加上数组的长度，再对数组进行取模

  3. Code

  1. Test cases

    MyCircularDeque circularDeque = new MycircularDeque(3); // set the size to be 3
    circularDeque.insertLast(1); // return true
    circularDeque.insertLast(2); // return true
    circularDeque.insertFront(3); // return true
    circularDeque.insertFront(4); // return false, the queue is full
    circularDeque.getRear(); // return 2
    circularDeque.isFull(); // return true
    circularDeque.deleteLast(); // return true
    circularDeque.insertFront(4); // return true
    circularDeque.getFront(); // return 4

Leetcode 622 Design Circular Queue(设计环形队列)

Posted on 2019-12-28 Edited on 2020-03-05 In leetcode

Difficulty : Medium

Type: Design, Queue

Language : Java

  1. Clarification

    Design your implementation of the circular queue. The circular queue is a linear data structure in which the operations are performed based on FIFO (First In First Out) principle and the last position is connected back to the first position to make a circle. It is also called “Ring Buffer”.

    One of the benefits of the circular queue is that we can make use of the spaces in front of the queue. In a normal queue, once the queue becomes full, we cannot insert the next element even if there is a space in front of the queue. But using the circular queue, we can use the space to store new values.

    Your implementation should support following operations:

    • MyCircularQueue(k): Constructor, set the size of the queue to be k.
    • Front: Get the front item from the queue. If the queue is empty, return -1.
    • Rear: Get the last item from the queue. If the queue is empty, return -1.
    • enQueue(value): Insert an element into the circular queue. Return true if the operation is successful.
    • deQueue(): Delete an element from the circular queue. Return true if the operation is successful.
    • isEmpty(): Checks whether the circular queue is empty or not.
    • isFull(): Checks whether the circular queue is full or not.

  2. Possible Solutions

    • 取余实现循环队列
      • 定义数组最大容量maxSize，队列头front和队列尾rear（默认初始值为0），以及用于存放队列的数组arr
      • Constructor 里设计maxSize = k + 1; 为的是预留一个空位，方便在尾部添加数据，数组长度为4的时候，实际最大容量是3（arr数组的长度为3）
      • 判断队列是否为空，只需判断front是否等于rear，如果等于，说明队列为空
      • 判断队列是否为满，判断 (rear + 1) % maxSize 是否等于 front，意思是，队列尾的下一个元素如果是队列头，队列就满了，由于rear可能大于maxSize，需要进行取余操作令rear指针在数组中循环
      • 入队操作，先判断队列是不是满了，满了就返回false，否则令arr[rear] = value，rear = (rear + 1) % maxSize，先把当前入队的值赋给数组，再让尾结点指向数组下一个位置（同样需要取模使其在数组里循环，这样走到maxSize - 1的位置时，下一个rear的位置就会回到0索引，实现循环），返回true
      • 出队列操作，先判断队列是不是空，空了就返回false，否则令front = (front + 1) % maxSize, 由于front也需要循环，同样需要对其进行取模处理
      • 获取队列头，先判断是不是空，是空返回-1，否则返回arr[front]即为队列头的值
      • 获取队列尾，先判断是不是空，是空返回-1，否则返回arr[(rear - 1 +maxSize) % maxSize]，因为rear在每次入队操作时都会指向下一个位置，所以需要让rear - 1，当rear = 0时可能出现数组越界（rear = -1 ），所以进行取模操作

  3. Code

  1. Test cases

    MyCircularQueue circularQueue = new MycircularQueue(3); // 设置长度为 3

    circularQueue.enQueue(1); // 返回 true

    circularQueue.enQueue(2); // 返回 true

    circularQueue.enQueue(3); // 返回 true

    circularQueue.enQueue(4); // 返回 false，队列已满

    circularQueue.Rear(); // 返回 3

    circularQueue.isFull(); // 返回 true

    circularQueue.deQueue(); // 返回 true

    circularQueue.enQueue(4); // 返回 true

    circularQueue.Rear(); // 返回 4

Leetcode 19 Remove Nth Node From End of List(删除链表的倒数第N个结点)

Posted on 2019-12-27 Edited on 2020-03-05 In leetcode

Difficulty : Medium

Type: Linked List, Two Pointers

Language : Java

  1. Clarification

    Given linked list 1 -> 2 -> 3 -> 4 -> 5, and n = 2.

    Remove the second node from the end and return the modified linked list.

  2. Possible Solutions

    • 快慢指针(Fast and slow pointers)
      • 先设置一个空的头结点dummy，指向head
      • 设置快慢指针fast和slow，让它们都处于dummy位置
      • 让fast指针先走n步，到达正数的第n个结点
      • 再让slow指针和fast指针同时每次走一步，直到fast.next = null，此时slow指针所在的位置就是倒数的第n - 1个结点位置
      • slow.next = slow.next.next，即删除倒数第N个结点，让倒数第n - 1个结点直接链到倒数第N个结点的下一个结点，返回dummy.next，即返回删除结点后的链表

  3. Code

  1. Test cases

    Input: 1 -> 2 -> 3 -> 4 -> 5, n = 2
    Output: 1 -> 2 -> 3 -> 5

Leetcode 203 Remove Linked List Elements(移除链表元素)

Posted on 2019-12-26 Edited on 2020-03-05 In leetcode

Difficulty : Easy

Type: Linked List

Language : Java

  1. Clarification

    Remove all elements from a linked list of integers that have value val\.

  2. Possible Solutions

    • Iteration(迭代)
      注意头节点情况，如果头节点需要被删除，需要把head结点移至head.next。设置两个结点prev和cur，cur指向head，prev为cur的前驱结点，删除时，prev.next -> cur.next，再令cur = cur.next即可，若当前遍历结点不需删除，prev指向cur，cur指向cur.next
    • Reversion（递归）
      • 将链表分解为头结点和一个更小的链表
      • 在更小的链表中删除给定的元素，将新链表res返回
      • 然后判断头结点应不应该被删除，如果应该被删除，就返回新链表res，如果不该被删除，就把新链表链在head后，返回head
      • 不能再小的基本问题是：链表缩短到不能再短，即链表中没有结点，此时head == null

  3. Code

  1. Test Cases

    Input: 1->2->6->3->4->5->6, val = 6
    Output: 1->2->3->4->5

Type: String, Dynamic Programming

Language : Java

  1. Clarification

    Given a string s, find the longest palindromic substring in s. You may assume that the maximum length of s is 1000.

  2. Possible Solutions

    • 暴力破解法（Brute Force）

      1. 枚举所有长度大于等于2的子串，依次判断它们是否为回文
      2. 此方法会超时，无法通过AC，可用于检验思路

    • 中心扩散法

      1. 回文串就是从中间开始往两边走，字符是轴对称的，根据此特性生成字符串，一共有2n-1个中心，奇数长度时中心个数是n个，偶数长度时中心个数是n-1个，所以一共2n-1个中心
      2. expandByCenter方法从中间点往两边找最长的回文串，比较长度，记录最长长度的start index和end index，作为返回substring的index

  3. Code

时间复杂度：O(n^3) 空间复杂度O(1)

  1. Test cases

    Input: “ababcba”
    Output: “abcba”

Leetcode 25 Reverse Nodes in k-Group(k个一组翻转链表)

Posted on 2019-12-18 Edited on 2020-03-05 In leetcode

Difficulty : Hard

Type: Linked list

Language : Java

  1. Clarification

    Given a linked list, reverse the nodes of a linked list k at a time and return its modified list. k is a positive integer and is less than or equal to the length of the linked list. If the number of nodes is not a multiple of k then left-out nodes in the end should remain as it is.

  2. Possible Solutions

    • Non-recursive

      Set a dummy code in the beginning and we need prev and last pointer. Last pointer will move k*1 distance. If k = 3, then last is 4. If last is null , then left nodes are smaller than k, which means these nodes are not supportive for next swap. Each iteration we reverse elements between prev and last (last and prev are excluded). For example, k = 3, we reverse elements from 1 to 3:

      1. We put 2 after prev and connects it to 1. Then linked list becomes prev -> 2 -> 1-> 3 -> last
      2. We put 3 after prev and connects it to 2. Then linked list becomes prev -> 3 -> 2 -> 1 -> last

Pointer tail will point to 1 in this case. ( Tail’s next node will always be Last! ). Tail will become the prev for next swap.

  1. Code

    Space Complexity : O(1)

  2. Test Cases

    Given this linked list: 1->2->3->4->5

    For k = 2, you should return: 2->1->4->3->5

    For k = 3, you should return: 3->2->1->4->5
