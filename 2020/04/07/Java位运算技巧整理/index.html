Java位运算技巧整理

2020-04-07 • 分类: Java • 标签: Bit Manipulation

刷题的时候老能看到一些大佬用我看不懂的位运算快速解题，感觉这个位运算自己要是整明白了应该做题能有更大进步，而且位运算较之取余操作是更快的，整理一些常用的位运算技巧和使用场景。

位运算操作符

  • & 与
    两个位都是1，结果才为1
  • | 或
    两个位都是0，结果才为0
  • ^ 异或
    两个位相同时位0，相异为1
  • ~ 取反
    0变1，1变0
  • << 左移
    各二进位全部左移若干位，高位丢弃，低位补0
  • >> 右移
    各二进位全部右移若干位，对无符号数，高位补 0，有符号数，各编译器处理方法不一样，有的补符号位(算术右移)，有的补 0 (逻辑右移)

技巧

判断奇偶数

只要根据最后一位是0还是1决定，0就是偶数，1就是奇数，可以用 if((a & 1)) == 0 代替 if(a % 2 == 0)来判断a是不是偶数。

交换两数
  • 第一步，a ^= b 即 a = (a ^ b)
  • 第二步，b ^= a 即 b = b ^ (a ^ b) = b ^ b ^ a（异或满足交换律，一个数和自己异或的结果为0且任何数与0异或都会不变，所以此时b被赋上了a的值）
  • 第三步，a ^= b就是a = a ^ b = (a ^ b) ^ a，a会被赋上b的值
变换符号

正数变负数，负数变正数，只需要取反操作后加1

求绝对值

对于负数，可以通过变换符号方式对其取反再加一得到正数，所以先移位来取符号位，int i = a >> 31;，注意如果a是正数，i等于0，为负数，i等于-1，然后对i进行判断，如果i等于0，直接返回。否则，返回~a + 1。

可以继续优化，因为任何数和0做异或都保持不变，与-1异或就相当于取反。所以，a与i异或后再减i（因为i为0或-1，所以减i即是要么加0要么加1）也可以得到绝对值。
