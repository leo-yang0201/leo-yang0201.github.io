Leetcode 239 Sliding Window Maximum(滑动窗口的最大值)

2020-03-26 • 分类: leetcode • 标签: Heap, Sliding Window

Difficulty : Hard

Type: Heap,Sliding Window

Language : Java

Clarification

  • Given an array nums, there is a sliding window of size k which is moving from the very left of the array to the very right. You can only see the k numbers in the window. Each time the sliding window moves right by one position. Return the max sliding window.

    Follow up:
    Could you solve it in linear time?

    Constraints:

    • 1 <= nums.length <= 10^5
    • -10^4 <= nums[i] <= 10^4
    • 1 <= k <= nums.length

Possible Solutions

  • 暴力法

    两层for循环暴力，每次都从窗口中找最大值即可

  • 优先队列
    每次循环都找最大的值，可以想到用优先队列构造一个最大堆，当优先队列的数组等于窗口大小的时候，只需要将第一个元素删除，然后将新的数字加入。由于最大堆的数据结构，堆的顶部就是当前最大值，只需要对优先队列进行peek操作就可以得到最大值。时间复杂度是O(nk)

  • 改良优先队列
    用TreeMap代替Queue，将时间复杂度降为log级别，为O(nlogK)

  • 单调队列
    以上三个方法的时间负责度都不是线性的，使用单调递减队列可以将时间复杂度降为O(n)。如果当前元素比队列的最后一个元素大，就将最后一个元素出队，重复此操作直到队列为空或者当前元素小于队列的最后一个元素。如果当前元素小于等于队列的最后一个元素或者队列为空，那么就直接讲当前元素入队。
    按照上述方法添加元素，队列中的元素就是一个单调递减的序列，最大值就是队头元素。当队列的元素等于窗口的大小时，因为添加元素时进行了出队操作，需要先判断一下队头元素是否是我们要删除的元素。
