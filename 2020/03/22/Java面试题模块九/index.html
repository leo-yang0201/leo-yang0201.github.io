冒泡排序

  • 算法描述

    • i从0开始，i与i+1比较，如果i>i + 1，那么就互换
    • i不断增加，直到i<n - 1（n是数组元素的个数，n - 1是数组最后一个元素），一趟下来，可以让数组元素中最大值排在数组的最后面

  • 冒泡算法优化

    • 如果在某趟排序中没有发生交换位置，那么我们可以认为该数组已经排好序了

  • 代码实现要点

    • 两个for循环，外层控制排序的趟数，内层控制比较的次数，每趟过后，比较的次数都应该要减1

  • 稳定排序算法

  • 时间复杂度：最好情况O(N)，最差情况O(N^2)，平均情况O(N^2)，空间复杂度O(1)

选择排序

  • 找到数组中最大的元素，与数组最后一位元素交换
  • 当只有一个数时，则不需要选择了，因此需要n - 1趟排序，比如10个数，需要9趟排序
  • 不稳定排序算法
  • 时间复杂度O(N^2)(所有情况下都是)，空间复杂度O(1)
  • 代码实现要点
    • 两个for循环，外层循环控制排序的趟数，内层循环找到当前趟数的最大值，随和与当前趟数组最后的一位元素交换

AOP利用一种横切的技术，将影响了多个类的公共行为封装到一个可重用模块，将其命名为”Aspect”切面，切面就是把和业务无关却为业务模块所共同调用的逻辑和责任封装起来，降低模块间的耦合度，并有利于未来的可操作性和可维护性。

使用“横切”技术，AOP把软件系统分为两个部分：核心关注点和横切关注点，业务处理的主要流程是核心关注点，与之关系不大的是横切关注点，横切关注点经常发生在核心关注点的多处，且各处基本相似。比如权限认证、日志、事务处理。AOP的作用在于分离系统中的各种关注点，将核心关注点和横切关注点分离开来。

插入排序

  • 将一个元素插入到已有序的数组中，在初始时未知是否存在有序的数据，因此将元素第一个元素看成是有序的
  • 与有序的数组进行比较，比它大则直接放入，比它小则移动数组元素的位置，找到个合适的位置插入
  • 当只有一个数时，则不需要插入了，因此需要n - 1趟排序，比如10个数，需要9趟排序
  • 稳定排序算法
  • 时间复杂度O(N^2)，最好情况O(N)，空间复杂度O(1)
  • 代码实现
    • 一个for循环内嵌一个while循环实现，外层for循环控制需要排序的趟数，while循环找到合适的插入位置（并且插入的位置不能小于0）

快速排序

  • 在数组中找一个元素（结点），比它小的放在节点的左边，比它大的放在节点右边。一趟下来，比结点小的在左边，比结点大的在右边。
  • 递归这个操作直到只剩一个元素
  • 快排处理过程是由上到下的，先分区，然后再处理子问题，是一个原地排序算法
  • 不稳定排序算法
  • 时间复杂度O(Nlog2N),最坏情况O(N)，空间复杂度O(nlog2n)

归并排序

  • 将两个已经排好序的数组合并成一个有序的数组
    • 将元素分割开来，看成是有序的数组，进行比较合并
    • 不断拆分和合并，直到只有一个元素
  • 稳定排序算法
  • 时间复杂度O(Nlog2N)(所有情况下都是)，空间复杂度O(n)，不是原地排序算法，处理过程是由下到上的，先处理子问题，然后再合并，性能上会占用更多内存
  • 代码实现
    • 在第一趟排序时实质是两个元素（看成是两个已经有序的数组）来进行合并，不断执行这样的操作，最终数组有序
    • 拆分左边，右边，合并

堆排序

  • 堆排序用到了完全二叉树的一个特性，根结点比左孩子和右孩子都要大，完成一次建堆的操作实质上是比较根结点和左孩子、右孩子的大小，大的交换到根结点上，直到最大的节点在树顶
  • 随和与数组最后一位元素进行交换
  • 代码实现
    • 只要左子树或右子树大于当前的根结点，则替换。替换后会导致下面的子树发生变化，因此同样需要进行比较，直至各个结点实现父>子这么一个条件

希尔排序

  • 希尔排序实际上是插入排序的增强版、希尔排序将数组分割成n组来进行插入排序，直至该数组宏观上有序，最后再进行插入排序时就不用移动太多次位置

  • 代码实现

    • 希尔增量一般是gap = gap / 2，只是比普通版插入排序多了这么一个for循环

基数排序（桶排序）

  • 将数组切割成个、十、百、千位放入到不同的桶子里，放一次就按桶子顺序回收一次，直到最大位数的数字放完
  • 稳定排序算法
  • 时间复杂度O(d(r + n))，最好情况O(d(n+rd))，空间复杂度O(rd+n)，r代表关键字的基数，d代表长度，n代表关键字的个数
  • 限制：
    • 桶排序需要很容易就能划分成m个桶，且桶与桶之间有天然的大小顺序。这样每个桶内的数据排序完后，桶与桶之间的数据不需要再进行排序
    • 数据在各个桶之间的分布是比较均匀的。如果有的桶里的数据非常多，有的非常少，那桶内数据排序时间的复杂度就不是常量级了，极端情况可能所有数据都被划分到一个桶里，就退化为O(nlogN)的排序算法
  • 适用场景：
    • 外部排序：数据存储在外部磁盘中，数据量大，内存有限，无法将数据全部加载到内存里
  • 代码实现
    • 先找到数组的最大值，然后根据最大值/10来作为循环的条件（只要>0，那么就说明还有位数）
    • 将个位、十位、…分配到桶里，每分配一次就回收一次
