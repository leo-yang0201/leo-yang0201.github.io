Hyperledger学习总结

Posted on 2020-07-06 In Blockchain Views:

基本概念（Hyperledger Fabric Concept)

  • DLT - Transactions - Chaincode

    • Distributed Ledger
      • 作用：Track the state of an asset
        Asset可以是有形的或者是无形的，如车辆、房屋、股票、证书等，Distributed Ledger记录这些asset的状态
    • Transactions
      • 作用：Manages the state of asset 管理asset状态
        • Invocation of the business logic 调用业务逻辑
        • Chaincode implements the logic 通过Chaincode实现逻辑
        • Recorded in the ledger 所有的Transaction都被记录在ledger中，这些record transaction是不可变的，即不能被更新或删除
    • Chaincode
      • Applications execute functions by way of Transactions 通过事务执行功能，如向区块链中存储一条记录等功能
      • Functions exposed for reading the state
      • 可用Go, NodeJS或Java编写Chaincode

  • Client side invocation(API)

    • Invoke 负责执行chaincode中的业务逻辑
    • Query 负责从distributed ledger读取asset状态
    • invoke和query都执行chaincode中的方法

    • invoke和query不同在于当你用invoke执行chaincode的方法时，transaction会被记录到ledger，而query执行chaincode方法时，transaction不会被记录，它只会读取asset状态并将其返回给caller。

    • 当用户调用invoke API，transaction proposal被创建并发送给Endorsing Peers，这些Peers再发回signed Transaction Proposal。然后，Client在把signed Tx Proposal发给Orderer，Orderer会建立Block(包含Transaction)并把block发给peers，并不一定每个peer都有chaincode，即使peer没有导致transaction生成的chaincode，它仍然可以接收有transaction的Block

    • 当用户调用query API， query function只会在有chaincode的peer上执行，client不能在没有chaincode的peer上执行query方法，client也不会和orderer进行交互，所以没有block会被生成

    基本概念总结

Chaincode 生命周期

  • CC Development
    • Specifications (Smart contract = Specifications)
      • Transaction interfaces
      • Business rules
      • Work Flow logic
    • Coding
      • Chaincode implements Smart Contract
    • Package
      • Put the package in a shared repository. Organizations can access this repo so admin from Orgs pick up the package to share code and install it on the peers in their respective organizations
      • Developer from that Orgs can implement the specifications on their own and clear or specific chaincode implementation(机构可以使用shared chaincode package，但也可以自定义自己的implementation，此时这个自定义的版本只有定义它的机构可使用)
  • Installation
    • Orgs install the chaincode
    • Generates the Package-ID; needed for approval(管理员可以用peer二进制文件安装package，如果写入正确，返回一个package id给peer二进制文件，用于approval)
    • Chaincode package id must be consistent within Org，否则不会被approved
  • Approval
    • Organizations approve the CC parameters
  • Committing
    • One of the Org Admin commits
