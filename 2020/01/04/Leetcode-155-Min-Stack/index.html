Leetcode 155 Min Stack(最小栈)

2020-01-04 • 分类: leetcode • 标签: Design, Stack

Difficulty : Easy

Type: Stack

Language : Java

  1. Clarification

    • Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.
      • push(x) – Push element x onto stack.
      • pop() – Removes the element on top of the stack.
      • top() – Get the top element.
      • getMin() – Retrieve the minimum element in the stack.

  2. Possible Solutions

    • 辅助栈

      因为要求常数时间内找到栈的最小值，采用“空间换时间”的思路，做一个辅助栈来存取最小值，另外一个数据栈存取数据。下面是一个图解（感谢windliang大佬的举例，简单易懂），大体思路就是新加入的元素如果大于栈顶元素，辅助栈就不处理，如果小于栈顶元素，就入辅助栈，数据栈出栈元素不等于辅助栈栈顶元素，辅助栈不操作，数据栈出栈元素等于辅助栈栈顶元素，栈顶元素都出栈。

      入栈 3
      | | | |
      | | | |
      |3| |3|
      stack minStack

      入栈 5 ， 5 大于 minStack 栈顶，不处理
      | | | |
      |5| | |
      |3| |3|
      stack minStack

      入栈 2 ，此时右边的 minStack 栈顶就保存了当前最小值 2
      |2| | |
      |5| |2|
      |3| |3|
      stack minStack

      出栈 2，此时右边的 minStack 栈顶就保存了当前最小值 3
      | | | |
      |5| | |
      |3| |3|
      stack minStack

      出栈 5，右边 minStack 不处理
      | | | |
      | | | |
      |3| |3|
      stack minStack

      出栈 3
      | | | |
      | | | |
      | | | |
      stack minStack

  • 用一个栈实现

    用一个变量而不是一个栈来存储最小值，需要解决一个问题，遇到更小的数值时，如何保存之前的最小值？本答案思路是提前把它在新的最小值之前压入栈里即可，当出栈元素是最小元素时，只需要把最小元素出栈，再出栈一次，把前一个最小元素赋值给min。

    通过以后方式，就只需要一个栈了，有更小的值来的时候，把之前的最小值入栈，当前更小的值入栈即可，当这个最小值要出栈的时候，下一个值便是之前的最小值了。

  • 差值解决存储最小值问题
    同样用一个min变量保存最小值，只不过不去保存原来的值，而是存储入栈的值和最小值的差值。若想得到之前的最小值，就可以通过min值和栈顶元素得到，举图例：
    入栈 3，存入 3 - 3 = 0
    | | min = 3
    | |
    |0|
    stack

    入栈 5，存入 5 - 3 = 2
    | | min = 3
    | 2 |
    |0|
    stack

    入栈 2，因为出现了更小的数，所以我们会存入一个负数，这里很关键
    也就是存入 2 - 3 = -1, 并且更新 min = 2
    对于之前的 min 值 3, 我们只需要用更新后的 min - 栈顶元素 -1 就可以得到
    | -1| min = 2
    | 2 |
    |0|
    stack

    入栈 6，存入 6 - 2 = 4
    | 4 | min = 2
    | -1|
    | 2 |
    |0|
    stack

    出栈，返回的值就是栈顶元素 4 加上 min，就是 6
    | | min = 2
    | -1|
    | 2 |
    |0|
    stack

    出栈，此时栈顶元素是负数，说明之前对 min 值进行了更新。
    入栈元素 - min = 栈顶元素，入栈元素其实就是当前的 min 值 2
    所以更新前的 min 就等于入栈元素 2 - 栈顶元素(-1) = 3
    | | min = 3
    | 2 |
    |0|
    stack

    此解法缺点在于保存的是差值，可能造成溢出，选用数据范围更大的long类型，优点在于每次更新最小值时没有存入最小值而是通过当前最小值和栈顶元素推出了之前的最小值，所以会省一些空间。
